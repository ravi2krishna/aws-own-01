    -> AWS Global Infrastructure 

        -> Amazon Cloud Computing resources like(Servers, Databases, Storage etc)
        are available across the world. 

        -> Global Infrastructure Map - https://aws.amazon.com/about-aws/global-infrastructure/

        -> The AWS Cloud spans 123 Availability Zones within 
        39 Geographic Regions, with announced plans for 7 more Availability Zones 
        and 2 more AWS Regions in the Kingdom of Saudi Arabia, and Chile.

        -> In Terms of AWS their infrastructure is broken down into

            -> Regions 

            -> Availability Zones 

            -> Edge Locations 

        -> Regions 

            -> Geographic Locations (Logical)

            -> Regions are designed to service AWS Customers that are Located 
            Closest to Region 

            -> Availability Of Services are different for different AWS Regions 

            -> "Security" is available Globally 

            -> Price Varies From Region To Region 

            -> NOTE: One Region is a combination of Multiple Availability Zones (AZ's)


        -> Availability Zones   

            -> An Availability Zones is where we have actual / Physical data center 

            -> Each Availability Zone has it's Independent Power Supply, Networking, Cooling
            System, Physical Security

            -> Multiple Availability Zones within a Region are Physically separated(100-150kms) 
            through Low Latency & High Speed Internet Connections 

            -> NOTE: Properly Designed Applications will always utilize Multiple 
            Availability Zones for High Availability & Fault Tolerance

            -> https://dta0yqvfnusiq.cloudfront.net/facti15983775/2019/08/AWS-Regions-and-AZs-5d4475fa4ec9d.jpg

        -> Edge Locations

            -> A Content Delivery Network (CDN) is a geographically distributed 
            group of servers that caches content (images, videos, HTML, JavaScript) 
            closer to end-users, reducing latency and accelerating page loads. 
            By delivering data from nearby "edge servers" rather than a single origin server.

            -> https://www.aalphanetsolution.com/blog/wp-content/uploads/2020/01/cdn_cover.png

            -> AWS Service for Providing CDN Capabilities is "Cloudfront" using Edge Locations


    -> Virtual Private Cloud - VPC 

        -> A  Virtual Private Cloud is a secured & isolated private cloud, 
        hosted within the public cloud.

        -> VPC is a service that lets customers(IBM) provision a 
        logically isolated section of the AWS Cloud where ibm can launch 
        AWS resources(Servers) in a VIRTUAL NETWORK that ibm define.

        -> NOTE: Having a vpc is similar to having your own data center inside AWS

        -> VPC is the Backbone of AWS Infrastructure

        -> NOTE: https://media.licdn.com/dms/image/v2/D4D12AQFI8sqvP4xFFQ/article-cover_image-shrink_720_1280/article-cover_image-shrink_720_1280/0/1700537758681?e=2147483647&v=beta&t=zE97TFGw5FQJEjUksqx7SohNsqqV57-IYQ_pj7K2M2U

    
    -> VPC Components

        -> Internet Gateway (IGW)

            -> Entry and Exit Point For (VPC)

            -> An IGW must be attached to a VPC if the resources inside the VPC need to 
            connect to resources via the open internet.

        -> Subnets 

            -> A Subnet ia logical subdivision of VPC 

            -> A Public Subnet can be reached from Internet

            -> A Private Subnet cannot be reached from Internet

        -> Route Tables 

            -> A route tables contains set of rules called Routes,
            which determine how the traffic to be directed.

            -> Public Route Tables contains internet based route

            -> Private Route Tables contains local / intranet route 

        -> Network Access Control Lists - NACL's 

            -> Subnet Level Firewalls 

            -> Subnet is group of Servers 

            -> By default everything is allowed inside subnet via NACL 

            -> NACL Have ALLOW / DENY Rules 

        -> Security Groups 

            -> Server Level Firewall

            -> Ever Server by default is ALL BLOCKED 

            -> section Groups ONLY ALLOW

    -> Subnetting

        -> Subnetting is the practice of dividing a network into two or more smaller networks.

        -> When you create a VPC, you must specify a CIDR (Classless Inter Domain Routing) block for the VPC.

        -> The allowed block size is between a /16 netmask (65,536 IP addresses) and 
        /28 netmask (16 IP addresses).

        -> AWS recommends to create a CIDR with 10.0.0.0/16 for future purpose

        -> The first four IP addresses and the last IP address in each subnet CIDR block are not available for you to use, and cannot be assigned to an instance.
            
            -> 10.0.0.0: Network address.
            -> 10.0.0.1: Reserved by AWS for the VPC router.
            -> 10.0.0.2: Reserved by AWS for Amazon DNS server.
            -> 10.0.0.3: Reserved by AWS for future use.
            -> 10.0.0.255: Network broadcast address.

    -> Server Management 

        -> Server Management is done using service called EC2 

        -> EC2 - Elastic Compute Cloud

        -> EC2 allows users to rent virtual computers (Servers/Machines) on which 
        "you can run your own computer applications"

        -> NOTE: Server in aws is referred as EC2 Instance / Instance

            -> Computer has hardware                        -> Server has Instance Type [cpu + ram]
                i3 / i5 / 4 GB RAM etc                          t2.micro, t2.medium, t2.large etc 

            -> Computer has hard disk                       -> Instance has EBS (Elastic Block) Storage
                250 GB / 500 GB / 1 TB                          250 GB / 500 GB / 1 TB                      
            
            -> Computer has network                         -> Instance has VPC 

            -> Compute has operating system                 -> Instance has AMI - Amazon Machine Image 

            -> Computer has Authentication                  -> Instance has Authentication
                username & password                             key pairs (public & private key)

            -> Computer has applications                    -> Instance has applications
                chrome / notepad etc                            myntra / flipkart / netflix etc
